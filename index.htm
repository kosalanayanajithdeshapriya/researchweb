Research at a Glance

Develop a dual-branch deep learning system that classifies tomato plant growth stages by using leaf density as the primary biological indicator — combining U-Net segmentation (Branch 01) with CNN-based classification (Branch 02) into a fused prediction model.

Why This Research?

Research Gap [3][5]

Existing deep learning work on tomatoes focuses on fruit detection and disease identification, NOT whole-plant growth stage classification. No study explicitly uses leaf density as a primary biological indicator for tomato stage detection.

Our Contribution

First dual-branch architecture that fuses pixel-level leaf segmentation (LPF) from U-Net with deep visual features from CNN to classify all four tomato BBCH growth stages.

Limitations of Existing Approaches [3][4][11]

ApproachWhat it MissesStudies
Manual inspectionLabor-intensive, subjective, not scalable for large greenhouse operations[1][2]
Classical ML (SVM, HOG)High intra-class similarity, no generalisation, manual feature engineering[11]

U-Net / LPF aloneLoses rich visual texture, shape, and colour information[14]
Our Dual-Branch FusionGets both: structural density + visual features[proposal]

Biological Justification [2][6]

Leaf Area Index (LAI) and leaf count increase systematically across tomato growth stages [6]. This means leaf density (measured as Leaf Pixel Fraction — LPF) is a reliable, quantitative biological signal for stage discrimination — not just a visual feature, but a structural one backed by agronomy literature [2].

Hyperparameters

Loss Functions

Branch 01 — Segmentation Branch (U-Net)

Role: Take a tomato RGB image → identify leaf pixels → compute Leaf Pixel Fraction (LPF). LPF is passed to the fusion layer as a quantitative biological density measure.

Complete Branch 01 Pipeline — 12 Steps

1
Original Tomato RGB Image
Original unmodified greenhouse photo (Xiaomi 12 Pro). Format: RGB, 3 channels. No annotation lines drawn on the actual image pixels. Captured at original full resolution.
2
Annotate Leaf Regions in Roboflow [4 (Roboflow)]
Open the original image in Roboflow, draw polygon outlines around all visible leaf regions using the Smart Polygon tool (semi-automated edge detection). The drawn polygon lines are saved separately as coordinate data — they do NOT modify the image pixels.
3
Export as COCO JSON
Annotations exported in COCO JSON format — a JSON file (not an image) storing: image_id, file_name, segmentation coordinates [x1,y1, x2,y2...], category_id ("leaf"), area, and bounding box. CRITICAL RULE: COCO JSON is NEVER fed as an image into U-Net. It is only used to generate binary masks.
{"annotations": [{"image_id": 1, "category_id": 1, "segmentation": [[50,30,80,20,110,40,90,70,60,65]]}]}
4
Convert COCO Polygons → Binary Mask
Read JSON, create blank array same size as original image, fill polygon regions with value 1 (leaf = white), rest stays 0 (background = black). Result: binary mask where every pixel position (x,y) aligns exactly with the original image. Binary because only 2 classes: leaf and background.
Mask pixel: 1 = leaf (white), 0 = background (black)
5
Resize Both to 224×224 Together [proposal]
WhatTarget SizeMethodWhy
Original RGB image224×224×3Bilinear interpolationSmooth, suitable for colour images
Binary mask224×224×1Nearest-neighborPreserves 0/1 values — no fractional values created
Critical rule: Always resize both together with the same spatial transform. If augmentation is applied, apply the exact same operation to both.
6
Create Training Pair (Locked Pair)
For every training sample, U-Net receives one fixed locked pair:
INPUT: tomato_001.jpg (224×224×3) — goes INTO U-Net
TARGET: tomato_001_mask.png (224×224×1) — what U-Net should OUTPUT
Every pixel at (x,y) in the image must align with the label at (x,y) in the mask.
7
U-Net Architecture — Internal Pipeline [14][Geeks]
PartSpatial SizeChannelsFunction
Encoder L1224→11264Basic edges, colours, textures
Encoder L2112→56128Leaf shapes, patterns
Encoder L356→28256Leaf vs background distinctions
Encoder L428→14512High-level structural features
Bottleneck14×141024Most compressed; knows "what" not "where"
Decoder L414→28—Upsample + merge skip features
Decoder L328→56—Refine leaf region boundaries
Decoder L256→112—Recover fine leaf shapes
Decoder L1112→224—Full resolution reconstruction

Skip Connections (most important feature of U-Net): Direct connections from each encoder level to the corresponding decoder level. They carry fine spatial details (exact leaf boundaries, small leaf shapes) that are lost during downsampling. Without skip connections, the output mask would have blurry, inaccurate leaf boundaries → less accurate LPF.

8
Output: Predicted Binary Leaf Mask
1×1 convolution + sigmoid activation: each pixel gets a value 0–1. Threshold at 0.5:
pixel > 0.5 → leaf (1, white) | pixel ≤ 0.5 → background (0, black)
At test time, only the image goes in — the model predicts the mask automatically for any unseen image.
9
Loss Computation and Training Update
Binary Cross-Entropy (BCE) loss:
BCE = -1/N × Σ [y·log(ŷ) + (1-y)·log(1-ŷ)]
where: y = true mask pixel, ŷ = U-Net predicted probability, N = total pixels
Loss backpropagated through all encoder, decoder, and bottleneck weights via Adam optimizer (lr=0.001). Repeats over many epochs.
10
Compute Leaf Pixel Fraction (LPF)
LPF = leaf_pixels / total_pixels = count_of_1s / (224 × 224)
Biological meaning across stages:
StageExpected LPFWhy
Seedling~0.05–0.15Very few leaves, sparse coverage
Vegetative~0.25–0.45Leaves growing, canopy forming
Flowering~0.55–0.75Full canopy, maximum leaf density
Fruiting~0.45–0.60Lower leaves begin senescence
11
Feature Fusion (passed to fusion layer)
LPF (scalar value) is concatenated with CNN feature vector from Branch 02 to form a 2049-dimensional combined vector.
12
Final Growth Stage Prediction
SoftMax outputs 4 probabilities → highest probability = final predicted BBCH growth stage.

The One Rule You Must Never Forget

COCO JSON is never the input to U-Net.
It is only used to create the binary mask ground truth.
The original image is always the input.
The binary mask is always the target output.
They must always be the same spatial size and the same matching pair.

Branch 02 — CNN Classification Branch

Role: Process the same 224×224×3 tomato image through a pre-trained CNN, extract a high-dimensional deep feature vector that captures visual patterns (leaf colour, texture, canopy structure), and pass it to the fusion layer.

What the CNN Learns [4][11]

Low-Level Features (Early Layers)

Basic edges, colour boundaries, simple textures. Detects where leaves start and background ends.

High-Level Features (Deep Layers)

Canopy texture, leaf colour patterns, structural shape relationships. Discriminates between visually similar stages like Vegetative vs Flowering.

Transfer Learning Strategy [4][11]

Why transfer learning? The dataset has only ~1,600 images — too small to train a deep CNN from scratch without overfitting. Pre-training on ImageNet (1.2M natural images) provides a strong starting point. Fine-tuning updates the weights to specialise for tomato leaf patterns. All images normalised using ImageNet channel mean and standard deviation.

CNN Architectures Being Compared [4][11]

ArchitectureFeature Dim.Strengths
ResNet502048Residual connections prevent vanishing gradients; strong for fine-grained visual differences
VGG164096Simple uniform architecture; strong baseline for plant classification
MobileNet1280Lightweight; good for deployment on edge devices in greenhouses

One objective of the research is to compare these architectures and select the best-performing one.

CNN Feature Vector Output

After the final pooling/flatten layer, the CNN produces a deep feature vector. Example with ResNet50:

Feature vector = [0.23, 0.87, 0.04, 0.65, 0.12, ...] ← 2048 numbers

Each number represents something the CNN learned about the image:
— leaf colour patterns, canopy texture, shape complexity, density gradients, etc.

The CNN does NOT need polygon annotations or masks. It only needs: the image + class label (Seedling/Vegetative/Flowering/Fruiting).

Image Preprocessing for CNN [proposal]

1
Capture Original Full-Resolution Image
Annotate leaf polygons on original image using Roboflow (for U-Net branch only). CNN branch does not need the annotations — it only needs the image and class label.
2
Resize to 224×224 (Bilinear Interpolation)
Neural networks require uniform input dimensions. Resize using bilinear interpolation. Apply consistently to ALL images across the dataset. If images are from the same phone with similar framing, simple resize to 224×224 is acceptable.
3
Normalize Using ImageNet Statistics [4][11]
mean = [0.485, 0.456, 0.406] (R, G, B)
std = [0.229, 0.224, 0.225] (R, G, B)
Ensures effective transfer learning — the model expects inputs in the same scale as ImageNet.
4
Data Augmentation (Training Only) [11][13]
Random horizontal flip, rotation, brightness/contrast adjustment, Gaussian blur. For CNN branch: augment only the image (no mask needed). This robustifies against lighting variation and plant positioning in the greenhouse.

Why Annotate at Original Resolution? [Roboflow]

Best practice answer for examiner: "We annotated the originally captured full-resolution images because higher-resolution images provide better leaf boundary detail during polygon annotation. After exporting annotations in COCO JSON format, we resized the images and their corresponding masks consistently to 224×224 during preprocessing. This preserves annotation quality while ensuring compatibility with both the CNN and U-Net input size requirements."

Feature Fusion — How Both Branches Unite

Step-by-Step Fusion Process

Branch 01 (U-Net): Image → Binary Mask → LPF = 0.65 (1 scalar)
Branch 02 (CNN): Image → Feature vector = [0.23, 0.87, ...] (2048 numbers)

Feature Fusion: Concatenate → [2048 CNN features | 0.65 LPF] = 2049-dim vector

FC Layer: 2049 → 256 (learns weights for both sources)

SoftMax: 256 → 4 probabilities (sum = 1.0)

Prediction: argmax(probabilities) → final growth stage

Training Flow — How Both Branches Learn Simultaneously

1
Forward Pass — Both Branches
Same image fed into both branches. U-Net produces binary mask prediction, CNN produces feature vector. Both processed simultaneously in one forward pass.
2
Loss Computation
U-Net Loss (BCE): Compare predicted leaf mask vs. true binary mask (from COCO annotations)
Classification Loss (Categorical Cross-Entropy): Compare predicted growth stage probabilities vs. true stage label
Two separate losses — each targets its own branch
3
Backpropagation
BCE loss backpropagates through U-Net (encoder, bottleneck, decoder). Classification loss backpropagates through CNN and the FC/SoftMax layers. Both update their own weights via Adam optimizer (lr=0.001). Both branches learn from the same training data at the same time.
4
Epoch Iteration
Steps 1–3 repeat for all training samples across many epochs until: predicted masks closely match true masks AND predicted growth stage probabilities are accurate. Validation performance monitored after each epoch to detect overfitting.
5
Inference (Test Time)
New unseen tomato image → same forward pass → no COCO JSON or ground truth needed → model automatically predicts: (1) leaf mask, (2) LPF, (3) CNN features, (4) final growth stage.

Fully Connected Layer — What It Learns

The FC layer learns to combine both information sources intelligently. Examples of what it discovers during training:

"When LPF is HIGH and CNN features show lots of green canopy texture → likely Flowering"
"When LPF is LOW and CNN features show few, sparse leaves → likely Seedling"
"When LPF is MEDIUM but CNN detects yellow/orange patterns → possibly Fruiting"

Model Evaluation Metrics [5][11]

Accuracy

Accuracy = (TP + TN) / Total Predictions

Overall proportion of correctly classified images across all 4 stages. Primary metric. Comparable to [11] which achieved 97.6% for wheat with multi-scale CNN.

Per-Class Precision

Precision = TP / (TP + FP)

For each stage: how often the model is correct when it predicts that stage. Important for Vegetative/Flowering boundary which is hardest to discriminate.

Per-Class Recall

Recall = TP / (TP + FN)

For each stage: how many actual instances of that stage are correctly detected. Shows if any stage is being missed/underdetected.

Confusion Matrix [5][11]

4×4 matrix showing all misclassifications. Particularly important for identifying Vegetative–Flowering confusion — the most visually ambiguous boundary in the dataset.

U-Net Specific Metrics [14]

IoU (Intersection over Union)

IoU = |predicted ∩ true| / |predicted ∪ true|

Measures how well the predicted leaf mask overlaps with the ground truth. [14] achieved 90% mean IoU for leaf segmentation in Brassica napus — comparable target for our work.

Dice Coefficient / F1

Dice = 2|P ∩ T| / (|P| + |T|)

Alternative segmentation quality metric. Related to IoU. Higher Dice = better mask accuracy = more reliable LPF computation.

Key Challenge: Vegetative–Flowering Boundary [5][11]

Why this is hard: During the Vegetative→Flowering transition, a plant simultaneously exhibits features of both stages. Leaf density (LPF) is at a mid-range value, and visual patterns are similar. This is where the dual-branch fusion is most beneficial — the FC layer can combine the LPF signal with CNN-detected flower buds or colour shifts to resolve the ambiguity.

Viva / Presentation Q&A — Click to Expand Answers

How do the CNN and U-Net produce the final prediction together?▼

"CNN and U-Net do not predict the growth stage independently. Instead, U-Net produces a binary leaf mask from which we compute Leaf Pixel Fraction as a scalar leaf-density value, while the CNN produces a high-dimensional deep feature vector from the same input image. These two outputs are concatenated in a feature fusion step, and the combined vector passes through a Fully Connected layer that learns how to weight both the visual and structural information together. A SoftMax activation then converts the output into four class probabilities — Seedling, Vegetative, Flowering, and Fruiting — and the highest probability becomes the final predicted growth stage."

Why use leaf density (LPF) as the key biological indicator?▼

"Biological evidence shows that Leaf Area Index and leaf count increase systematically across tomato growth stages [2][6]. This means LPF — measured as the fraction of leaf pixels in the image — changes predictably from seedling (sparse, ~0.08) through vegetative and flowering (dense, ~0.65) and slightly declines at fruiting. This makes LPF a reliable, quantitative structural signal that complements the CNN's visual features, particularly for resolving ambiguous stage boundaries."

Why is COCO JSON never fed into U-Net?▼

"COCO JSON is a metadata file — not an image. It stores polygon coordinates describing where leaves are in an image. U-Net expects a pixel array (image tensor) as input, not a JSON file. The COCO JSON is only used during preprocessing to generate binary mask images, which are then used as the ground truth targets during training. The actual input to U-Net is always the original RGB image."

Why use nearest-neighbor interpolation for masks but bilinear for images?▼

"The binary mask only contains values 0 or 1. Bilinear interpolation creates intermediate values (e.g. 0.4, 0.7) by blending neighbouring pixels — this would introduce non-integer values into the mask, destroying the binary labels the model needs. Nearest-neighbor interpolation always snaps to the closest original pixel value, preserving the strict 0/1 binary nature of the mask."

What are skip connections in U-Net and why are they important?▼

"Skip connections are direct shortcuts from each encoder level to its corresponding decoder level. During downsampling (the encoder path), fine spatial details like exact leaf boundaries and small leaf shapes are progressively lost as the image is compressed. Skip connections carry those fine details directly to the decoder, allowing it to reconstruct the mask with precise boundaries. Without skip connections, the output mask would be blurry and inaccurate — leading to less reliable Leaf Pixel Fraction computation."

Why annotate on the original full-resolution image, not the resized 224×224?▼

"Annotating at full resolution is better because: (1) original images have more pixel detail, making it easier to accurately trace leaf boundaries with the Smart Polygon tool; (2) higher resolution reduces annotation errors at leaf edges. After exporting in COCO JSON format, both the images and their corresponding masks are resized consistently to 224×224 during preprocessing. The polygon coordinates are scaled together with the image, so the masks remain spatially aligned. This is acceptable and technically correct practice."

Why use transfer learning rather than training from scratch?▼

"Our dataset has only ~1,600 images, which is too small to train a deep CNN from scratch without severe overfitting [4][11]. Transfer learning using ImageNet pre-trained weights provides a strong starting point — the model already knows how to detect edges, textures, and shapes from millions of natural images. Fine-tuning then specialises these weights for tomato leaf patterns. This significantly reduces overfitting and allows the small dataset to achieve strong performance."

How are both branches trained simultaneously?▼

"During each training iteration, the same image is fed into both branches in a single forward pass. Two losses are computed: Binary Cross-Entropy (BCE) for the U-Net segmentation branch comparing the predicted mask against the true binary mask, and Categorical Cross-Entropy for the final classification comparing predicted stage probabilities against the true stage label. Both losses backpropagate through their respective branches independently, updating the U-Net weights and CNN weights separately via Adam optimizer. This means both branches learn from the same training data at the same time, in a joint end-to-end training process."

What is the hardest classification challenge in this research?▼

"The most challenging boundary is between the Vegetative and Flowering stages [5][11]. During this transition, a plant can simultaneously exhibit features of both stages — the canopy is fully formed but flowers may only be beginning to emerge. Both the visual appearance and the LPF value are at mid-range values. The dual-branch architecture helps here: even if the CNN visual features are ambiguous, the FC layer can leverage the LPF value combined with any detected colour or texture changes to resolve the boundary more accurately than either branch alone."

What is the BBCH scale and why do you use it?▼

"The BBCH (Biologische Bundesanstalt, Bundessortenamt und CHemische Industrie) scale is a standardized numerical system for describing plant growth stages from germination through maturity [7]. It is widely adopted for tomato phenology documentation in research and agriculture. Using BBCH codes ensures our stage definitions are scientifically grounded and comparable to existing literature — our four stages (Seedling BBCH 00-13, Vegetative 15-19, Flowering 60-69, Fruiting 70-79) align precisely with established biological milestones."
